# Fase 0.5 — Skeleton (Spec)

**Data:** 2026-05-07
**Fase anterior:** F0 (monorepo bootstrap, plano `2026-05-07-monorepo-bootstrap.md` — concluído).
**Próxima fase:** F1 (Auth, spec `2026-05-07-fase-1-auth.md`).

---

## 1. Objetivo

Estabelecer a **base compartilhada** que todas as features verticais consomem: shadcn/ui + paleta + fonts no Next, Prisma + helmet + throttler + validator + healthcheck no Nest, Postgres local via docker-compose com setup automatizado, e deploy reproduzível no Railway via Dockerfiles.

A F0.5 **não entrega valor de usuário sozinha** — entrega infraestrutura. Ao fim:

- `npm install && cp .env.example apps/web/.env.local && cp .env.example apps/api/.env && npm run db:setup && npm run dev` sobe o ambiente local completo.
- Web em `:3000` mostra placeholder com tema dark e fonts aplicadas.
- API em `:3001` responde `GET /health` com 200 (validando conexão com Postgres).
- Deploy Railway: web + api + Postgres no ar.
- Sem auth, sem domínio. F1 começa a popular.

---

## 2. Premissas (entregue por F0)

A F0.5 **não** instala/configura nada do que segue — assume tudo pronto:

- Monorepo `npm workspaces` + Turborepo, layout `apps/web` (Next.js 16, App Router, Tailwind v4), `apps/api` (NestJS 11), `packages/shared-types`.
- Tooling dia 0: ESLint, Prettier (com `prettier-plugin-tailwindcss`), Husky + lint-staged (auto-format no pre-commit), commitlint (conventional commits), `.editorconfig`, `.nvmrc` (Node 22).
- GitHub Actions com lint + typecheck + build em PRs e push para `main`.
- Repositório no GitHub (`origin/main`), CI verde.

Se algum desses pontos não estiver pronto, **a F0.5 está bloqueada** e o gap volta pra F0.

### 5.4 Validação de env (zod)

`src/config/env.schema.ts`:

```ts
import { z } from 'zod';

export const envSchema = z.object({
  NODE_ENV: z.enum(['development', 'production', 'test']).default('development'),
  PORT: z.coerce.number().default(3001),
  DATABASE_URL: z.string().url(),
  ALLOWED_ORIGINS: z.string().min(1, 'ALLOWED_ORIGINS é obrigatório'),
});

export type Env = z.infer<typeof envSchema>;

export function validateEnv(raw: Record<string, unknown>): Env {
  const parsed = envSchema.safeParse(raw);
  if (!parsed.success) {
    throw new Error(
      `Env inválida:\n${parsed.error.issues.map((i) => `  - ${i.path.join('.')}: ${i.message}`).join('\n')}`,
    );
  }
  return parsed.data;
}
```

### 5.5 Bootstrap (`src/main.ts`)

Substituir o `main.ts` que veio do scaffold por:

```ts
import { NestFactory } from '@nestjs/core';
import { ValidationPipe, Logger } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import helmet from 'helmet';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  const config = app.get(ConfigService);
  const logger = new Logger('Bootstrap');

  app.use(helmet());

  app.useGlobalPipes(
    new ValidationPipe({
      whitelist: true,
      forbidNonWhitelisted: true,
      transform: true,
    }),
  );

  app.enableCors({
    origin: config
      .getOrThrow<string>('ALLOWED_ORIGINS')
      .split(',')
      .map((s) => s.trim()),
    credentials: false,
  });

  const port = config.get<number>('PORT') ?? 3001;
  await app.listen(port);
  logger.log(`API running on http://localhost:${port}`);
}

void bootstrap();
```

### 6.4 `scripts/db-setup.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail

CONTAINER_NAME="invoices-postgres"
TIMEOUT_SEC=60

echo "→ subindo Postgres..."
docker compose up -d postgres

echo "→ aguardando healthcheck (timeout ${TIMEOUT_SEC}s)..."
ELAPSED=0
while true; do
  STATUS=$(docker inspect --format='{{.State.Health.Status}}' "$CONTAINER_NAME" 2>/dev/null || echo "starting")
  if [ "$STATUS" = "healthy" ]; then
    echo "✓ Postgres healthy"
    break
  fi
  if [ "$ELAPSED" -ge "$TIMEOUT_SEC" ]; then
    echo "✗ Postgres não ficou healthy em ${TIMEOUT_SEC}s (status atual: $STATUS)"
    exit 1
  fi
  sleep 2
  ELAPSED=$((ELAPSED + 2))
done

echo "→ rodando prisma migrate deploy..."
npm --workspace=@invoices-ocr/api exec -- prisma migrate deploy

if node -e "process.exit(require('./apps/api/package.json').prisma?.seed ? 0 : 1)" 2>/dev/null; then
  echo "→ rodando prisma db seed..."
  npm --workspace=@invoices-ocr/api exec -- prisma db seed
else
  echo "→ sem seed configurado, pulando"
fi

echo "✓ db:setup completo"
```

`chmod +x scripts/db-setup.sh` — versionado executável.

## 12. Sequência sugerida de implementação

A spec da F0.5 não impõe ordem, mas a sequência abaixo minimiza retrabalho ao executar via writing-plans:

1. **DevOps mínimo** (docker-compose + scripts + .env.example) — sem isso nada roda.
2. **Backend** (Prisma init + PrismaModule + bootstrap + health + módulos vazios) — depende de Postgres pra smoke do `/health`.
3. **Frontend** (shadcn init + globals.css + fonts + providers + página placeholder + topbar estático) — independente do backend.
4. **Dockerfiles + railway.json** — depende de FE+BE buildando local primeiro.
5. **Deploy Railway** — depende de Dockerfiles validados.
6. **README final + smoke prod + CI verde**.

Steps 2 e 3 podem rodar em paralelo se houver mais de uma pessoa ou em PRs separados.
