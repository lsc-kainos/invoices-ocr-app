# F2.5 — OCR Schema Evolution + Benchmark Admin (Spec)

**Data:** 2026-05-09  
**Fase anterior:** F2 (OCR pipeline, spec `2026-05-07-fase-2-ocr.md` + adições `2026-05-09`).  
**Próxima fase:** F3 (Chat — spec já existente, com emendas desta F2.5).

---

## 1. Objetivo

Duas entregas acopladas:

1. **Schema OCR universal**: substituir o `core` BR-focado por campos universais de invoice (alinhados aos fixtures sintéticos de benchmark), adicionar `items[]` e um campo `narrative` (resumo em linguagem natural). Campos BR específicos (CNPJ, chave NF-e, CFOP) migram para `extras[]`. A mudança impacta o extractor prompt, o Zod schema, a UI rail e os system prompts do chat (F3).

2. **Benchmark admin**: serviço que lê o dataset `samples/invoice-dataset/` (imagens + CSVs com ground truth), roda o `OpenAiOcrProvider` real em cada imagem, compara os campos extraídos contra o ground truth via fuzzy match, e expõe os resultados em tempo real via SSE num endpoint admin. Página `/admin/benchmark` no frontend permite acionar e acompanhar.

---

## 2. Premissas

- F2 completa: `OcrModule`, `OpenAiOcrProvider`, `invoiceSummarySchema`, `DocumentsModule`, `StorageModule` no ar.
- Dataset em `samples/invoice-dataset/`: fixtures sintéticos versionados (`batch1-XXXX.jpg`) + CSVs de ground truth (`index.csv` e splits de compatibilidade).
- `User.role` enum no schema Prisma (F1) inclui os valores `USER` e `ADMIN`. Se não existir o valor `ADMIN`, adicionar via migration antes da F2.5.
- `JwtAuthGuard` global com `@Public()` opt-out (F2). `UserScopedThrottlerGuard` como `APP_GUARD`.
- Atribuição do role `ADMIN` a um usuário: manual via Prisma Studio ou seed de desenvolvimento. Sem UI de gestão de roles nesta fase.

---

## 3. Decisões fechadas

| #   | Decisão                  | Escolha                                                                                                                  |
| --- | ------------------------ | ------------------------------------------------------------------------------------------------------------------------ |
| D1  | Schema `core`            | Universal (campos do CSV sintético). BR fields → `extras[]` via instrução no prompt                                      |
| D2  | Campo `narrative`        | String livre, gerada na mesma chamada OpenAI estruturada. 2-4 frases descrevendo a nota                                  |
| D3  | `items[]` no schema      | Array tipado dentro de `summary`, separado do `core`. Campos: `description`, `quantity`, `unitPrice`, `totalPrice`       |
| D4  | Benchmark lifecycle      | Ferramenta pontual (não CI). Acionada via página admin. Resultados não persistem em DB                                   |
| D5  | Execução do benchmark    | SSE streaming: 1 evento por imagem + 1 evento `complete` com agregado                                                    |
| D6  | Comparação de campos     | Fuzzy match após normalização (remove `$`, `,`, espaços, lowercase). Match exato pós-normalização                        |
| D7  | Proteção do endpoint     | `RolesGuard` + `@Roles('ADMIN')`. Guard lê `req.user.role`                                                               |
| D8  | Reutilização do provider | Benchmark importa `OpenAiOcrProvider` diretamente (sem DI container). Mesmo prompt, mesmo schema                         |
| D9  | Retrocompatibilidade     | `narrative` é gerado na F2.5 em diante. Docs pré-existentes sem `narrative` usam fallback `JSON.stringify(core)` no chat |
| D10 | F3/F4 impacto            | F3: `buildWorkspaceSystem` e `buildDocumentSystem` usam `narrative`. F4: adiciona `narrative.txt` opcional no ZIP        |

## 5. Schema Zod — `invoiceSummarySchema` (substituição)

Arquivo: `apps/api/src/ocr/schemas/invoice-summary.schema.ts`

```ts
import { z } from 'zod';

export const invoiceCoreSchema = z.object({
  invoiceNumber: z.string().nullable(),
  invoiceDate: z.string().nullable(), // ISO 8601 ou formato original da nota
  dueDate: z.string().nullable(),
  sellerName: z.string().nullable(),
  sellerAddress: z.string().nullable(),
  clientName: z.string().nullable(),
  clientAddress: z.string().nullable(),
  tax: z.string().nullable(), // string para preservar formato
  discount: z.string().nullable(),
  total: z.string().nullable(),
  paymentMethod: z.string().nullable(),
});

export const invoiceItemSchema = z.object({
  description: z.string(),
  quantity: z.string().nullable(),
  unitPrice: z.string().nullable(),
  totalPrice: z.string().nullable(),
});

export const invoiceExtraSchema = z.object({
  label: z.string(),
  value: z.string(),
  mono: z.boolean().optional(),
});

export const invoiceSummarySchema = z.object({
  summary: z.object({
    core: invoiceCoreSchema,
    items: z.array(invoiceItemSchema),
    extras: z.array(invoiceExtraSchema),
    narrative: z.string(),
  }),
  extractedText: z.string(),
});

export type InvoiceCore = z.infer<typeof invoiceCoreSchema>;
export type InvoiceItem = z.infer<typeof invoiceItemSchema>;
export type InvoiceSummary = z.infer<typeof invoiceSummarySchema>['summary'];
```

**Campos BR em `extras[]`:** o extractor prompt instrui o modelo a adicionar, quando presentes:

- `{ label: "CNPJ Emitente", value: "XX.XXX.XXX/XXXX-XX", mono: true }`
- `{ label: "CNPJ Destinatário", value: "...", mono: true }`
- `{ label: "Chave NF-e", value: "<44 dígitos>", mono: true }`
- `{ label: "CFOP", value: "5102" }`
- `{ label: "Tipo", value: "NF-e" }`

### 7.2 `csv-loader.ts`

Lê `samples/invoice-dataset/batch1_{1,2,3}.csv`. Cada linha produz:

```ts
type BenchmarkSample = {
  filename: string; // ex.: "batch1-0001.jpg"
  groundTruth: {
    invoiceNumber: string | null;
    invoiceDate: string | null;
    dueDate: string | null;
    sellerName: string | null;
    clientName: string | null;
    tax: string | null;
    total: string | null;
  };
  items: Array<{ description: string; quantity: string; totalPrice: string }>;
  ocrText: string; // "OCRed Text" do CSV — referência textual
};
```

O CSV usa vírgulas como delimitador com campos entre aspas; o JSON aninhado em `Json Data` é parseado via `JSON.parse`. Campos ausentes/vazios viram `null`.

### 7.3 `field-comparator.ts`

```ts
function normalize(v: string | null): string {
  if (!v) return '';
  return v
    .toLowerCase()
    .replace(/[$,\s]/g, '') // remove $, vírgulas, espaços
    .trim();
}

function matchField(extracted: string | null, expected: string | null): boolean {
  if (!expected) return true; // campo não disponível no ground truth → não penaliza
  return normalize(extracted) === normalize(expected);
}
```

Score por imagem: `camposCorretos / camposPresentes` (onde "presente" = ground truth não-null).

### 7.4 `benchmark.service.ts`

```ts
async *runStream(samplesDir: string): AsyncGenerator<BenchmarkEvent> {
  const samples = await loadCsvSamples(samplesDir);
  const total = samples.length;

  for (let i = 0; i < total; i++) {
    const sample = samples[i];
    const imagePath = join(samplesDir, sample.filename);

    try {
      const buffer = await fs.readFile(imagePath);
      const result = await this.provider.extract(buffer, 'image/jpeg');
      const parsed = invoiceSummarySchema.parse(result);
      const fieldResults = compareFields(parsed.summary.core, sample.groundTruth);
      const score = computeScore(fieldResults);

      yield {
        type: 'progress',
        index: i + 1,
        total,
        filename: sample.filename,
        score,
        fieldResults,
        narrative: parsed.summary.narrative,
      };
    } catch (err) {
      yield {
        type: 'progress',
        index: i + 1,
        total,
        filename: sample.filename,
        error: classifyError(err),
      };
    }
  }

  yield { type: 'complete', aggregate: computeAggregate(/* accumulated */) };
}
```

Imagens processadas **sequencialmente** (não em paralelo) para não exceder rate limit da OpenAI.

### 7.5 `benchmark.controller.ts`

```ts
@Controller('api/v1/admin')
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles('ADMIN')
export class BenchmarkController {
  constructor(private readonly benchmark: BenchmarkService) {}

  @Post('benchmark')
  @Throttle({ default: { ttl: 3600_000, limit: 5 } }) // 5 runs/hora por admin
  async run(@Res() res: Response): Promise<void> {
    res.setHeader('Content-Type', 'text/event-stream');
    res.setHeader('Cache-Control', 'no-store');
    res.setHeader('Connection', 'keep-alive');
    res.setHeader('X-Accel-Buffering', 'no');

    const samplesDir = join(process.cwd(), '../../samples/invoice-dataset');

    for await (const event of this.benchmark.runStream(samplesDir)) {
      res.write(`data: ${JSON.stringify(event)}\n\n`);
    }

    res.write('data: [DONE]\n\n');
    res.end();
  }
}
```

### 7.6 `RolesGuard`

```ts
// apps/api/src/auth/roles.guard.ts
@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(ctx: ExecutionContext): boolean {
    const roles = this.reflector.getAllAndOverride<string[]>('roles', [
      ctx.getHandler(),
      ctx.getClass(),
    ]);
    if (!roles) return true;
    const { user } = ctx.switchToHttp().getRequest();
    return roles.includes(user?.role);
  }
}
```

`@Roles()` é um decorator simples: `SetMetadata('roles', roles)`.

### 8.3 `use-benchmark.ts`

```ts
export function useBenchmark() {
  const [results, setResults] = useState<ImageResult[]>([]);
  const [aggregate, setAggregate] = useState<Aggregate | null>(null);
  const [running, setRunning] = useState(false);
  const [progress, setProgress] = useState({ current: 0, total: 0 });

  const run = useCallback(() => {
    setRunning(true);
    setResults([]);
    setAggregate(null);

    const es = new EventSource('/api/admin/benchmark', { withCredentials: true });

    es.onmessage = (e) => {
      if (e.data === '[DONE]') {
        es.close();
        setRunning(false);
        return;
      }
      const event = JSON.parse(e.data);
      if (event.type === 'progress') {
        setProgress({ current: event.index, total: event.total });
        setResults((prev) => [...prev, event]);
      } else if (event.type === 'complete') {
        setAggregate(event.aggregate);
      }
    };

    es.onerror = () => {
      es.close();
      setRunning(false);
    };
  }, []);

  return { run, running, progress, results, aggregate };
}
```

**Decisão SSE no cliente:** usar `fetch` + `ReadableStream` manual (consistente com o padrão de streaming do F3). `EventSource` não suporta `POST` nem headers customizados — e a rota Next precisa injetar o `Authorization: Bearer` no request ao Nest. O Next route handler `/api/admin/benchmark` é `POST`, o cliente chama com `fetch('/api/admin/benchmark', { method: 'POST' })` e lê `res.body` como `ReadableStream`.
