# R3: Chat com LLM

> Requisito: "Integrate a Language Model (e.g., GPT-4) to explain or provide context for the extracted text to the users. Enable user queries about extracted information."

---

## Visão Geral

O chat permite que o usuário converse sobre o conteúdo extraído de seus documentos. Existem dois modos:

1. **Chat por Documento:** pergunta sobre um documento específico
2. **Chat Workspace:** pergunta sobre todos os documentos do usuário

**Arquitetura:**

```
Usuário envia mensagem
    │
    ▼
ChatService monta contexto (resumos dos documentos)
    │
    ▼
OpenAI GPT-4o com function calling
    │
    ├── Se precisar de detalhes → get_full_document(id)
    │   (tool call para buscar texto completo)
    │
    ▼
Resposta persistida e retornada
```

## 2. Modos de Chat

### 2.1 Chat por Documento

```typescript
// apps/api/src/chat/chat.service.ts
async sendDocumentMessage(userId: string, documentId: string, content: string) {
  // Verifica se documento existe e está READY
  const doc = await this.prisma.document.findFirst({
    where: { id: documentId, userId },
  });
  if (!doc) throw new NotFoundException();
  if (doc.status !== 'READY') throw new ConflictException({ code: 'document_not_ready' });

  // Cria ou reusa sessão por (userId, documentId)
  const session = await this.prisma.chatSession.upsert({
    where: { userId_documentId: { userId, documentId } },
    create: { userId, documentId },
    update: {},
  });

  // Persiste mensagem do usuário
  await this.prisma.chatMessage.create({
    data: { sessionId: session.id, role: 'USER', content },
  });

  // Monta contexto e envia para LLM
  const result = await this.runConversation({
    userId,
    systemPrompt: buildDocumentSystem(llmConfig.prompt, doc),
    messages: history,
    persist: (msg) => this.prisma.chatMessage.create({ data: { sessionId: session.id, ...msg } }),
  });

  return { content: result.content };
}
```

### 2.2 Chat Workspace

```typescript
async sendWorkspaceMessage(userId: string, sessionId: string, content: string) {
  // Busca até 50 documentos READY do usuário
  const docs = await this.prisma.document.findMany({
    where: { userId, status: 'READY' },
    select: { id: true, filename: true, summary: true },
    orderBy: { updatedAt: 'desc' },
    take: 50,
  });

  const result = await this.runConversation({
    userId,
    systemPrompt: buildWorkspaceSystem(llmConfig.prompt, docs),
    messages: history,
    persist: (msg) => this.prisma.chatMessage.create({ data: { sessionId, ...msg } }),
  });

  return { content: result.content };
}
```

**Contexto:** resumos estruturados dos documentos (não o texto completo). Isso economiza tokens e mantém o contexto gerenciável.

## 3. Function Calling

### 3.1 Tool: `get_full_document`

```typescript
// apps/api/src/chat/tools/get-full-document.tool.ts
export class GetFullDocumentTool {
  static readonly name = 'get_full_document';

  static readonly schema: ChatCompletionTool = {
    type: 'function',
    function: {
      name: GetFullDocumentTool.name,
      description: 'Obtém o texto completo extraído de um documento',
      parameters: {
        type: 'object',
        properties: {
          documentId: { type: 'string', description: 'ID do documento' },
        },
        required: ['documentId'],
      },
    },
  };

  async execute(args: unknown, ctx: { userId: string }): Promise<unknown> {
    const { documentId } = args as { documentId: string };
    const doc = await this.prisma.document.findFirst({
      where: { id: documentId, userId: ctx.userId },
      select: { extractedText: true },
    });
    if (!doc) return { error: 'Document not found' };
    return { documentId, extractedText: doc.extractedText };
  }
}
```

**Quando é usada:** quando o LLM precisa de detalhes que não estão no resumo.

## 4. Loop de Conversação

```typescript
// apps/api/src/chat/chat.service.ts
private async runConversation(ctx: RunContext): Promise<{ content: string }> {
  const maxIter = this.config.get<number>('CHAT_MAX_TOOL_ITERATIONS') ?? 3;

  const conversation: ChatCompletionMessageParam[] = [
    { role: 'system', content: ctx.systemPrompt },
    ...ctx.messages.map(toOpenAiMessage),
  ];

  let iter = 0;
  while (iter < maxIter) {
    const resp = await this.llm.complete({
      model,
      messages: conversation,
      tools: this.tools.getOpenAiSchemas(),
      stream: false,
      temperature,
    });

    const message = resp.choices[0].message;
    const functionCalls = message.tool_calls?.filter(c => c.type === 'function') ?? [];

    if (functionCalls.length) {
      // Persiste tool call do assistant
      await ctx.persist({ role: 'ASSISTANT', content: '', toolCallId: functionCalls[0].id, ... });

      // Executa a tool
      for (const call of functionCalls) {
        const handler = this.tools.getHandler(call.function.name);
        const output = await handler(args, { userId: ctx.userId });

        // Persiste resultado da tool
        await ctx.persist({ role: 'TOOL', content: JSON.stringify(output), ... });

        // Adiciona ao contexto da conversação
        conversation.push({ role: 'tool', tool_call_id: call.id, content: JSON.stringify(output) });
      }

      iter++;
      continue;  // ← volta para o LLM com o resultado da tool
    }

    // Resposta final (sem tool calls)
    await ctx.persist({ role: 'ASSISTANT', content: message.content ?? '' });
    return { content: message.content ?? '' };
  }

  throw new InternalServerErrorException({ code: 'tool_loop_exceeded' });
}
```

**Loop:**

1. Envia contexto para o LLM
2. Se o LLM pedir uma tool → executa → volta ao passo 1
3. Se o LLM responder diretamente → retorna
4. Máximo 3 iterações (previne loops infinitos)

## 5. Histórico de Mensagens

### 5.1 Carregamento

```typescript
private async loadHistory(sessionId: string, userId: string) {
  const max = this.config.get<number>('CHAT_MAX_HISTORY') ?? 20;
  const all = await this.prisma.chatMessage.findMany({
    where: { sessionId, session: { userId } },
    orderBy: { createdAt: 'desc' },
    take: max,
  });
  const ordered = all.reverse();

  // Garante que o slice começa em USER
  // Se o limite cortar no meio de um tool exchange, o TOOL fica órfão
  // e a OpenAI rejeita com 400
  const firstUser = ordered.findIndex((m) => m.role === ChatRole.USER);
  return firstUser === -1 ? [] : ordered.slice(firstUser);
}
```

### 5.2 Segurança do Contexto

- **Ownership:** `session: { userId }` garante que só o dono vê as mensagens
- **Sanitização:** o conteúdo do documento vai como dado (entre `<documento>`), nunca como instrução
- **Limitação:** máximo 20 mensagens no contexto (configurável)

## 6. System Prompts

### 6.1 Chat por Documento

```typescript
// apps/api/src/chat/prompts/system.prompt.ts
export function buildDocumentSystem(rules: string, doc: ...) {
  const narrative = doc.summary?.narrative ?? '';
  const structured = {
    core: doc.summary?.core,
    items: doc.summary?.items ?? [],
    extras: doc.summary?.extras ?? [],
  };
  return [
    rules,
    '',
    'Contexto:',
    'O usuário está olhando um único documento. O resumo abaixo já inclui um sumário em prosa (narrative) e o JSON estruturado (core/items/extras). Use a tool apenas se isso não bastar.',
    '',
    `<document id="${escXml(doc.id)}">`,
    `  <filename>${escXml(doc.filename)}</filename>`,
    `  <narrative>${escXml(narrative)}</narrative>`,
    `  <summary>${escXml(JSON.stringify(structured))}</summary>`,
    `</document>`,
  ].join('\n');
}
```

### 6.2 Chat Workspace

```typescript
export function buildWorkspaceSystem(rules: string, docs: ...) {
  const list = docs.map((d) => {
    const narrative = escXml((d.summary?.narrative ?? '').toString().slice(0, 240));
    const core = escXml(JSON.stringify(d.summary?.core ?? {}));
    return `  <document id="${escXml(d.id)}" filename="${escXml(d.filename)}">
    <narrative>${narrative}</narrative>
    <core>${core}</core>
  </document>`;
  }).join('\n');

  return [
    rules,
    '',
    'Documentos do usuário (use get_full_document(documentId) para detalhe):',
    `<documents>`,
    list,
    `</documents>`,
  ].join('\n');
}
```

### 6.3 Prompt Base (LlmConfig)

O prompt system é versionado e configurável via admin (`/admin/llm-configs`). O padrão instrui o LLM a:

- Responder apenas em português brasileiro
- Usar os dados dos documentos como fonte única
- Nunca inventar informações
- Formatar valores monetários adequadamente
